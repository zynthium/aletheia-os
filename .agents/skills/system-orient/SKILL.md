# System Orient Skill

Use before any non-trivial task.

Read:
1. `aletheia_os/00_CHARTER.md`
2. `aletheia_os/02_ACTIVE_STATE.md`
3. relevant node(s) from `aletheia_os/01_SYSTEM_GRAPH.yaml` or `aletheia_os/nodes/`

Return:
- active node;
- parent constraints;
- current objective;
- non-objectives;
- success criteria;
- invalidation criteria;
- required durable updates;
- checkpoint plan.

If the active node cannot be identified, ask for a narrower task or propose a provisional node and mark it uncertain.

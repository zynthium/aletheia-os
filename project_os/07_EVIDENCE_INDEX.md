# Evidence Index

Evidence can be experimental, simulated, mathematical, operational, market-based, manufacturing-based, or customer-based. The point is not to collect facts; the point is to update the system graph.

## Evidence strength levels

| Level | Meaning |
|---:|---|
| 0 | Anecdote, intuition, or untested idea. |
| 1 | Plausibility check or toy example. |
| 2 | Controlled test/simulation/derivation with known limitations. |
| 3 | Robust cross-condition evidence or independently reproducible result. |
| 4 | Field/prod/live/physical evidence under realistic constraints. |
| 5 | Repeated high-quality evidence with monitoring and failure-boundary knowledge. |

## Interpretation rule

Every evidence record must state:

1. claim tested;
2. method;
3. result;
4. limitations;
5. graph impact;
6. next action.

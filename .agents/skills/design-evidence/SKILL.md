name: design-evidence

---

# Design Evidence Skill

Never design confirmation-only evidence.

Always include:
- claim tested;
- alternative explanations;
- method;
- controls;
- realism constraints;
- failure modes;
- evidence strength target;
- interpretation limits;
- expected graph impact if positive;
- expected graph impact if negative;
- required evidence record path.

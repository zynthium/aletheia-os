# Skeleton Review

Use this playbook to review whether the main truth tree still reflects the
project's best current structure.

## Review Questions

1. Does every active node still serve its parent?
2. Do sibling nodes have clear boundaries?
3. Are too many children hanging directly under one parent?
4. Has any leaf grown enough to become a branch?
5. Do multiple leaves imply a missing intermediate branch?
6. Are any active branches contradicted by evidence?
7. Are any incubating orphans ready to attach, split, merge, or archive?
8. Are root-level changes being proposed with human confirmation?

## Outputs

Use existing truth records:

- update `SKELETON.yaml` for small structural corrections;
- update node records for local boundary changes;
- create or update a decision for meaningful tree refactors;
- create or update evidence when the review depends on observations;
- update `ORPHANS.yaml` for incubator disposition.

## Review Cadence

Review the skeleton when:

- validation reports tree warnings;
- a handoff contains a candidate skeleton;
- multiple tasks touch the same node boundary;
- a branch accumulates conflicting evidence;
- the project profile changes toward scientific, quantitative, AI, or hybrid research.

# Playbook — Design Evidence

Evidence design must avoid confirmation-only tests.

Include:

- claim tested;
- null or alternative explanations;
- method;
- control conditions;
- failure modes;
- realism constraints;
- interpretation boundaries;
- graph impact if positive;
- graph impact if negative;
- stop or rebalance criteria.

Domain examples:

- Quant: leakage checks, OOS, transaction costs, capacity, regimes.
- Physics: limiting cases, dimensional checks, numerical convergence, comparability to known results.
- Aircraft/vehicle: simulation fidelity, safety margins, manufacturability, test procedures.
- Market/product: customer segment validity, competitive response, unit economics, channel constraints.

# START HERE

This repository uses AletheiaOS. Read `.aletheia/START_HERE.md` before non-trivial work.

Before durable changes, orient on the truth tree:

1. Identify the root objective.
2. Identify the active node and parent constraints.
3. Decide whether new material attaches to the main tree, stays in `.aletheia/state/ORPHANS.yaml`, or requires tree refactor.
4. For claims or hypotheses, identify evidence and invalidation criteria.
5. For implementation, identify the contract, node, or decision it serves.

For stable-node claims, also run `python3 .aletheia/bin/history_audit.py --json`.
Stability is only traceable after validation, evidence, decision, human
confirmation, and Git history audit agree.
